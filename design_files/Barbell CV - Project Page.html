01 / 05 Detection

Why wrists?

YOLO Pose wrist keypoint tracking
overlay on deadlift footage · GIF

YOLO Pose tracks 17 body keypoints per frame. Wrists grip the bar and move with it near-exactly — a stable proxy without needing to detect the barbell itself. Larger model sizes (large vs. nano) gave meaningfully more stable wrist tracks, with no inference-speed cost for offline video.

Confidence thresholding per keypoint means the pipeline degrades gracefully when occlusion happens — it flags rather than guesses.

▶ What I tried first
YOLO (direct) ✗
Bounding box jumped frame-to-frame and misfired onto equipment. Heavy post-processing still left it unreliable.
SAM2 ✗
Appealing idea — point once, track through the set. Drifted on real gym footage: variable lighting and cluttered backgrounds were enough to lose it.
YOLO Pose → wrists ✓
Pose models are trained on millions of humans in every orientation — vastly more robust than object-tracking on a barbell-shaped object in a gym environment.
02 / 05 Smoothing

Preserving velocity peaks

Raw vs. smoothed wrist position signal
concentric phases highlighted · plot

Raw keypoint positions jitter frame-to-frame — noise that gets amplified when differentiated into velocity. Savitzky-Golay fits a polynomial to a sliding window rather than averaging, so it preserves peak shape. A moving average would systematically underestimate peak bar speed: exactly the metric VBT depends on.

03 / 05 Phase Detection

Isolating the concentric phase

▶ How PELT changepoint detection works

PELT (Pruned Exact Linear Time) segments a signal into statistically distinct constant-mean pieces. A penalty term controls how many segments are allowed.

Applied to position, a single upward pull looks like many small constant-mean pieces — PELT over-segments. Applied to velocity, a clean concentric phase is a single near-zero-mean segment (bar moving at roughly steady speed) — exactly one piece for PELT to find.

Three post-processing steps follow the raw PELT output: merge adjacent candidates above velocity threshold, extend phase boundaries while velocity supports it, and reject candidates below 50% of median candidate velocity.

04 / 05 Output

Per-rep velocity

Per-rep velocity (px / sec) across a working set
bar chart · one bar per rep, trend visible

Output is px/sec — a relative metric. Valid as a within-session fatigue signal without calibration. Converting to m/s requires a checkerboard reference or known range-of-motion estimate; this is noted as an open problem.
